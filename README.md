# 谐波 · 实时嗓音分析器

实时采集麦克风信号，分析人声的**谐波结构（H1–H5）**，并把它翻译成嗓音训练里关心的听感维度：**气声 / 压声**、**音色明暗**、**频谱倾斜**。完整复现参考设计稿，产品级界面，纯前端、零依赖、本机处理。

![界面](docs/preview.png)

## 功能

- **实时基频检测（F0）**：YIN 算法（累积均值归一化差函数 + 抛物线插值），亚采样精度，带清晰度判定与静音门限；F0 经 5 帧中值滤波消除倍频误检。
- **谐波电平提取**：在 8192 点 FFT 谱上以紧窗（±1–3 bin）定位真正的谐波峰并抛物线插值，避免把噪声/泄漏误当成微弱高次谐波而抬高 H3–H5。
- **嗓音指标**
  - `H1-H2`：气声 / 压声（间隙越大越带气声）
  - `H1-H5`：整体衰减幅度
  - `低频/高频平衡` = mean(H3–H5) − mean(H1–H2)
  - `频谱倾斜`（dB/oct）：谐波电平对 log₂(频率) 的最小二乘斜率 —— **驱动「音色明暗」的核心量**
- **基频实时曲线**：独立画布显示最近 8 秒的音高轨迹，自适应 Hz 范围，曲线连续，末端实时标注当前基频；可在设置里开启**固定目标频率参考线**（练声对照）。
- **二维特征垫**：横轴「更深色 ↔ 更明亮」（由频谱倾斜驱动）、纵轴「已按压 ↔ 更轻柔」（由 H1–H2 驱动），数据点随发声实时移动，并由一条粉色包络曲线与柱状图相连。
- **动态文案**：顶部状态药丸与三张摘要卡的标题随指标实时变化（帧率无关的时间常数平滑，避免抖动）。
- **设置面板**：平滑度、静音门限、基频范围、目标基频参考线、参考基线开关、画面冻结，自动保存于本机。
- **隐私**：音频只在浏览器本地处理，不联网、不上传。

## 音色明暗的声学依据

「明暗」不是拍脑袋的方向，而是钉在声学事实上的：

- **频谱质心（spectral centroid）是感知亮度最稳健的相关量**——频谱重心越偏高频越明亮（McAdams 2019；Schubert & Wolfe 2006, JASA）。
- **频谱倾斜 / 斜率**：声门源谐波典型衰减约 **−12 dB/oct**；声门闭合越强 → 高次谐波越多 → **谱斜越平缓 → 高频能量越多 → 越明亮**；越陡 → 越暗。
- 因此本应用用 **`频谱倾斜 tilt`** 驱动明暗：以 −12 dB/oct 为中性基线，越接近 0 越亮、越负越暗（特征垫横轴同理）。
- 面向 **MtF/FtM 嗓音训练**：更明亮（更高质心 / 更平谱斜 / 更高共振峰与 CoG）= 女性化感知方向，反之偏男性化。

> 注：`低频/高频平衡(high−low)` 不适合判明暗——人声该值几乎恒为负（谐波本就向下衰减），会导致"明亮"永不触发；明暗一律以频谱倾斜为准。

## 共鸣方向（偏男 / 中性 / 偏女）的依据

共鸣性别方向由共振峰 **F1、F2** 决定，并钉在文献数据上，而非主观设定：

- **F1、F2 越高 → 越偏女**（声道越短 / 共鸣越靠前）；F2 是感知性别最强的共振峰相关量，F1 次之。
- 用 **Hillenbrand et al. (1995)** 成人男/女 12 元音共振峰数据校准：在与共鸣图一致的归一化空间里（F2 700–2900、F1 250–1050），F1 与 F2 对性别的分离度相当，故**等权**综合：
  `s = 0.5·norm(F2) + 0.5·norm(F1)`
- 男/女类心约 **0.355 / 0.469**，边界取中点 **≈0.41**；中性带 **[0.38, 0.44]**。`s<0.38` 偏男、`>0.44` 偏女。
- 录音报告的「共鸣方向占比」与共鸣空间图上的两条分界虚线**用的是同一组阈值**。
- 实测两个示例：male.wav → 偏男 91%、female.wav → 偏女 55%。逐帧仍有重叠（如 /u/、/o/ 等低共振峰元音帧会落入偏男区）——这是真实声学现象（Hillenbrand 中同一元音男女也部分重叠），并非误差。

## 运行

麦克风（`getUserMedia`）需要**安全上下文**。`http://localhost` 被浏览器视为安全，所以本地起一个静态服务器即可，无需 HTTPS。

```bash
cd voice-harmonics
python serve.py
```

脚本会在 `http://localhost:8000/` 启动并自动打开浏览器。也可以：

```bash
python serve.py 8080      # 指定端口
python serve.py --no-open # 不自动打开
# 或者用标准库
python -m http.server 8000
```

打开后点击「开始分析」，允许麦克风权限，对着麦克风发声即可。

> 直接双击 `index.html`（`file://`）无法获取麦克风——必须通过 `http://localhost`。

## 项目结构

```
voice-harmonics/
├── index.html        # 界面结构
├── styles.css        # 样式（深色玻璃质感 + 跨性别蓝/粉渐变强调 / 移动端优先）
├── serve.py          # 本地静态服务器（localhost 安全上下文）
└── src/
    ├── app.js        # 主控：音频 → DSP → 渲染 → 界面，设置与主循环
    ├── audio.js      # 麦克风采集与 Web Audio 管线（谱分析器 + 时域分析器）
    ├── dsp.js        # 基频检测、噪声基底、谐波提取、指标计算
    ├── interpret.js  # 指标 → 中文听感描述、特征垫坐标
    └── render.js     # Canvas 绘制：柱状图、特征垫、连接曲线
```

## 数据流

```
麦克风 ──► AudioEngine.poll() ──► { 频谱dB, 时域 }
                                      │
              detectPitch(时域) ──► F0、清晰度、RMS
                                      │  (有清晰人声时)
   extractHarmonics(频谱dB, F0) ──► H1..H5 电平(dB)
                                      │
                  computeMetrics ──► H1-H2 / H1-H5 / 低高平衡 / 倾斜
                                      │
         指数滑动平均平滑 + 渲染 ──► 柱状图 / 特征垫 / 药丸 / 摘要卡
```

差值类指标（H1-H2、H1-H5、低高平衡、倾斜）只依赖谐波间的相对电平，因此与噪声基底偏移无关，结果稳定。

## 浏览器支持

Chrome / Edge / Firefox / Safari 等现代浏览器（需支持 Web Audio API 与 ES Modules）。建议使用外接麦克风以获得更干净的谐波结构。
